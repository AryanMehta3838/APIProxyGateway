# UI Rules

- Keep the interface clean, readable, and uncluttered.
- Preserve responsive behavior.
- Follow the current design language unless instructed otherwise.
- Do not create display-only UI that is not actually wired to real state.
- Handle happy path first, then empty/loading/error states.
- Preserve or improve accessibility where possible.
- Avoid unnecessary visual complexity.
- Do not break navigation or back-flow behavior.
