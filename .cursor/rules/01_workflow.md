# Workflow Rules

- Work in small, testable increments.
- Before coding, identify the smallest viable slice of the requested change.
- Prefer minimal diffs over large rewrites.
- Reuse existing patterns before inventing new ones.
- After each meaningful change, validate it.
- Never claim something works unless it was actually tested or built.
- At the end of each task, summarize:
  - what changed
  - files changed
  - validation performed
  - remaining risks or next steps
- If the task is large, split it into substeps and do the first correct one.
